## null과 undefined

`null`과 `undefined` 모두 **'값이 없다'**&#xB77C;는 의미를 담고 있지만, 그 쓰임새는 조금씩 차이가 존재한다.

`undefined`는 자바스크립트 엔진이 변수 초기화 시 자동으로 할당하는 값이다. 반면 `null`은 개발자가 의도적으로 **'값이 없음'**&#xC744; 표현하기 위해 할당하는 값이다.&#x20;

이 둘은 느슨한 비교(`==`)에서는 같게 취급되지만, 엄격한 비교(`===`)에서는 다르게 취급된다.

<Callout variant="note">
  느슨한 비교는 두 값의 타입이 다르더라도 자동으로 형 변환을 진행한 후 둘의 값을 비교하지만 엄격한 비교에서는 데이터 타입까지 정확하게 일치해야 참으로 판정한다는 차이점이 있다
</Callout>

## 부동 소수점의 연산 오차

자바스크립트에서 `0.1 + 0.2 === 0.3`의 결과는 `false`이다. 그 이유는 부동소수점 형식의 한계에 있다.

자바스크립트에서 모든 숫자는 IEEE 754 표준을 따르는 64비트 부동소수점 형식으로 저장된다. 그러나 이 방식은 일부 소수점 연산에서 정확한 값을 저장하지 못하는 한계를 갖는다.&#x20;

예를 들어 0.1과 0.2는 이진수로 변환하면 무한 반복되는 소수로 표현되므로, 컴퓨터는 이를 근사값으로 저장한다. 이로 인해 결과적으로 `0.1 + 0.2`는 `0.30000000000000004`가 되는데, 이는 `0.3`과 일치하지 않기 때문이 `===` 연산자 사용 시 `false`가 된다.

이를 해결하기 위해서는 아래와 같이 두 가지 해결 방법이 있다.

1\. `toFixed`나 `toPrecision`함수를 이용해 특정 자릿수까지 반올림하는 방법. 단, 숫자가 문자열로 변환된다는 단점이 있다.

2\. `Number.EPSION`을 활용해 비교하는 방법. `Number.EPSION`은 JS에서 가장 작은 숫자이다.

```ts
function isEqual(a, b) {
  return Math.abs(a - b) < Number.EPSILON;
}

console.log(isEqual(0.1 + 0.2, 0.3)) // true;
```

<IdeographicSpace />

<Callout variant="note">
  이런 오차에도 불구하고 64비트 부동 소수점 연산을 사용하는 이유는 성능과 표현의 범용성 때문이다. 먼저 현대의 대다수 CPU에서 IEEE 754 연산을 하드웨어적으로 지원해서 매우 빠른 연산이 가능하다. 둘째로, 64비트라는 제한된 메모리 공간 안에서 정수와 실수를 구분하지 않고 아주 작은 소수부터 매우 큰 정수까지 폭 넓게 표현할 수 있어 설계가 단순해지고 활용도가 높아진다.
</Callout>

## 매개변수 전달 방식

**Call by \***&#xB294; <u>함수를 호출할 때 매개변수를 어떤 방식으로 전달하느냐</u>를 의미하는 용어이다.

### Call by Value

함수를 호출할 때 값 자체를 복사해서 전달하는 방식. JS에서는 원시 타입이 Call by Value로 전달된다.

### Call by Reference

함수를 호출할 때 값이 저장된 메모리 주소(참조값)를 전달하는 방식.

### Call by Sharing

함수를 호출할 때 값이 저장된 메모리 주소(참조값)를 복사해서 전달하는 방식. JS에서 객체와 같은 참조 타입이 이와 같은 방식으로 전달된다.

```ts
function updateTeam(count, members, leader) {
    // 1. 원시 타입(숫자) 재할당 시 이는 복사된 값이기 때문에 영향을 미치지 않음
    count = 20; 

    // 2. 참조 타입(배열)의 내부 요소 수정 시 원본 주소값은 유지한 채 내부 요소를 수정하는 것이기 때문에 원본 값에 영향을 미침.
    members.push('Park'); 

    // 3. 참조 타입(객체) 자체를 새로운 객체로 재할당하는 것은 매개변수가 가리키는 주소값만 변경할 뿐 원본값이 가리키는 주소값은 변하지 않아 원본값에 영향을 미치지 않음.
    leader = { name: 'Choi', role: 'Senior' }; 
}

let memberCount = 10;
let memberList = ['Lee', 'Kim'];
let teamLeader = { name: 'Jung', role: 'Junior' };

updateTeam(memberCount, memberList, teamLeader);

console.log(memberCount); // 10
console.log(memberList);  // ['Lee', 'Kim', 'Park']
console.log(teamLeader);  // { name: 'Jung', role: 'Junior' }
```
