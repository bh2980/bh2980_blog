## 문제

jQuery 처럼 속성을 chaining해서 설정할 수 있는 wrapper `$`를 만들어주세요.

```js
$('#button')
  .css('color', '#fff')
  .css('backgroundColor', '#000')
  .css('fontWeight', 'bold')
```

## 풀이

<Collapsible>
  ```js
  function $(el) {
    const element = el

    return {
  // @mark Tooltip {9-17} content="this 바인딩을 위한 일반 함수 호출"
      css: function(propertyName, value) {
  // @mark Tooltip {20-32} content="동적으로 객체 키를 설정하기 위해 대괄호를 통한 객체 키 접근"
        element.style[propertyName] = value;

        return {
  // @mark Tooltip {8-21} content="이 때 this는 function 키워드를 통해 호출한 객체를 이어 받는다."
          css: this.css
        };
      }
    }
  }
  ```

    ```mermaid
    graph TD;
      A["$(el) 단독 호출"] --> B["$ 실행 컨텍스트 생성"];

      C --> D["return { css: ... } 평가"];
      D --> E{"css 함수 종류?"};
      E -- "화살표 함수" --> F["css(arrow) 함수 객체 생성"];
      F --> G["상위 컨텍스트($)의 this 캡처/고정 (= undefined)"];
      G --> H["receiver = $(el)의 반환 객체"];
      H --> I["receiver.css(...) 호출"];
      I --> J["receiver를 this로 전달 시도"];
      J --> K["arrow는 호출 this를 받지 않음"];
      K --> L["캡처된 this(= undefined) 사용"];
      E -- "일반 function" --> M["css(function) 함수 객체 생성 (this 캡처 없음)"];
      M --> N["receiver = $(el)의 반환 객체"];
      N --> O["receiver.css(...) 호출"];
      O --> P["엔진이 this = receiver로 설정 후 실행"];
    ```

    1. $(el) 호출
    2. $의 **실행 컨텍스트(execution context)** 가 생성되고, 단독 호출이라 $의 this가 undefined로 결정(대부분 strict 환경)
    3. 화살표 함수
       1. return \{ css: (...) } 객체 리터럴을 **평가하는 순간** css의 **화살표 함수 객체가 생성**되고, 그때 상위 실행 컨텍스트($)의 this(이때는 undefined)를 **캡처**해서 고정
       2. $(el).css('color', '#fff') 실행 시 엔진은 메소드 호출 형태이므로 css에 $(el)의 반환 객체를 this로 지정하려고 시도하지만, 화살표 함수에는 호출로부터 this를 받는 매커니즘이 없어서 무시된다.
    4. function 함수
       1. css객체는 생성되지만 this는 고정하지 않는다. 이는 호출 시점에 결정된다.
       2. $(el).css('color', '#fff')를 호출하는 순간 엔진이 $(el)의 반환 객체를 css의 this로 지정하여 css 함수를 실행한다.
